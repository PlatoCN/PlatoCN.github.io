QGis二次开发-QGis源码编译QGIS3.4.15 + VS2017 + qt5.12 + win10

​ 关于QGis各种文件的理解参考这位老哥QGis二次开发 – 源码编译终极篇_MrJGao的博客-CSDN博客_qgis for c#

一、准备工作

  • 1、下载QGIS源码

    https://github.com/qgis/QGIS,最好用git clone到本地,这样可以看到所有的版本,缺点是clone的速度非常慢

  • 2、编译所需工具

CMake >= 2.8.0

Flex

Bison >= 2.4

默认安装即可

  • 3、第三方库

以下依赖库可直接在OSGeo4w上安装,安装后将OSGeo4W添加到系统变量里,具体请百度

Qt >= 4.5.0

Proj >= 4.4.x

GEOS >= 3.0

Sqlite3 >= 3.0.0

GDAL/OGR >= 1.4.x

Qwt >= 5.0 & (< 6.1 with internalQwtPolar)

expat >= 1.95

二、CMAKE配置

​ 第一个是源码路径,第二个是要新建的build,编译后的文件在此.

​ 点击configure,设置第一行VS2017,第二行x64 其余默认 会弹窗报错说项目无效.不要慌按照如下勾选

​ 开始可能会没有编辑这两个,再次点击configure后就会出现.

​ 干掉test可以加快编译速度.任性的同学请随意

​ 编译如果不通过,编码也要改一下,如果不行,openMP换成MP

​ Win10可能在文件里找不到SetupAPI.lib文件,被隐藏了.直接搜就行

​ 依赖库的设置参照这位老哥QGIS编译_箜_Kong的博客-CSDN博客_windows编译qgis

​ 后点击Generate.出现如下 便成功了

​ 点击Open Project打开项目,如下操作
